# `User`
***
### api/user
#### GET - получить всех пользователей
> проверка авторизации - нет
Возвращаемые значение
| Код | Описание |
| ---- | ----------|
| 200 | масив объектов пользователей или [] |
Пример ответа
```json
[{"id": 1,
"firstName": "Olga",
"middleName": "Petrovich",
"lastName": "Ivanova", (or null)
"email": "olga@test.ua",
"settings": "text" (or null),
"roleIds": [1,2],
"statusId": 1,
"creationDate": "2019-07-23T11:57:20.695102+03:00"}]
```
***
### /api/user?id=[id]
#### GET - получить пользователя по id
> проверка авторизации - нет
Передаваемые параметры
| Название | Тип | Описание |
| ---- | ----------| -------|
| id | `int` | id пользователя |
Возвращаемые значение
| Код | Описание |
| ---- | ----------|
| 200 | объект пользователя с этим id |
| 204 | No Content - пользователя с таким id не существует |
Пример ответа
```json
{"id": 1,
"firstName": "Olga",
"middleName": "Petrovich",
"lastName": "Ivanova", (or null)
"email": "olga@test.ua",
"settings": "text" (or null),
"roleIds": [1,2],
"statusId": 1,
"creationDate": "2019-07-23T11:57:20.695102+03:00"}
```
***
### /api/user
#### POST - регистрация нового пользователя с ролью учителя или любыми другими ролями
> проверка авторизации - нет
Передаваемые параметры в теле запроса ( row body )
`json object`
| Поле | Тип | Описание |
| ---- | ----------| -------|
| firstName | `string` | обязательное поле длиной от 2 до 50 символов |
| middleName |`string` | поле длиной от 2 до 50 символов |
| lastName | `string` | обязательное поле длиной от 2 до 50 символов |
| email | `string` | обязательное поле формата email |
| settings | `string` | обязательное поле длиной от 2 до 50 символов |
| roleIds | `array` | если не передать, присвоится роль Teacher |
| password |`string` | обязательное поле не менее 6 символов, включая нижний, верхний регистры и цифры |
Пример запроса
```json
{ "firstName": "Olga",
"middleName": "Petrovich",
"lastName": "Ivanova",
"email": "olga@test.ua",
"settings": "text",
"roleIds": [1,2],
"password": "1asdfASD" }
```
Возвращаемые значение
| Код | Описание | Пример |
| ---- | ----------| --- |
| 200 | пользователь и его роли сохранены в базе |`{"Saved": ""}`|
| 400 |ошибка валидации даных | `{"email": "Value can't be null!", "middleName": "Length have been > 1 and < 50"}` |
| 400 | Такой email уже есть в базе даных | `{"Error": "Email not unique"}` |
| 400 | В масиве ролей не существующая роль | `{"Error": "No such role"}` |
